## Log command in Docker Compose:
- **Comando:** `docker compose up`
- **Descrição:** Este comando inicia todos os serviços definidos no arquivo `docker-compose.yaml` em segundo plano. Ele cria e inicia os contêineres, redes e volumes necessários para o funcionamento do projeto.
- **Saída esperada:** O comando deve exibir logs de inicialização dos serviços, incluindo o coletor OpenTelemetry, o Zipkin e os dois serviços Go. A saída deve incluir mensagens de log indicando que os serviços estão prontos para receber requisições.
- Execução não apresenta erros ao executar o build do service goapp.
